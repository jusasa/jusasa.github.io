객체지향 설계 및 인터페이스

클래스를 통한 데이터 캡슐화부터, 인터페이스를 활용한 다형성 구현까지 객체지향의 핵심을 다룹니다. 단순한 변수 접근(필드)을 넘어 속성(Property)과 인덱서(Indexer)를 통해 데이터를 안전하게 관리하고, 구현부가 없는 '기능의 약속'인 인터페이스가 어떻게 유연한 코드 구조를 만드는지 탐색할 수 있습니다.

속성 (Property) & 인덱서

왜 속성을 사용하는가?

  • 데이터 보호 (캡슐화) 및 제어
  • 잘못된 값 입력 방지 (예: 나이가 음수)
  • 읽기 전용(`get`), 쓰기 전용 구현 가능
private int age;
public int Age 
{
  get { return age; }
  set { 
    if(value < 0) return; 
    age = value; 
  }
}

인터페이스 (Interface)

인터페이스의 특징

  • 클래스가 반드시 구현해야 하는 기능의 규약
  • 구현부 없음 (선언만 존재), 객체 생성 불가(`new` 불가)
  • 다중 상속 가능
  • 다형성 구현의 핵심 (예: 무기 교체 시스템)
interface IWeapon { 
  void Attack(); 
}

class Sword : IWeapon {
  public void Attack() {
    Console.WriteLine("칼로 공격!");
  }
}

데이터 구조: 배열(Array) 분석

동일한 자료형의 데이터를 체계적으로 관리하는 배열의 종류와 특징을 비교합니다. 단순한 목록형 데이터(1차원)부터 표 형태의 데이터(2차원), 그리고 복잡한 계층형 데이터(가변 배열)의 메모리 할당 및 선언 방식을 차트와 탭 인터페이스를 통해 효율적으로 파악할 수 있습니다.

1차원 배열 (1D Array)

가장 기본적인 형태. 하나의 선(Line) 구조로 데이터를 저장합니다.

int[] numbers = new int[5];
0
1
2
3

2차원 배열 (2D Array)

행(Row)과 열(Column)로 구성된 표(Table) 구조. 모든 행의 길이가 동일합니다.

int[,] matrix = new int[2, 3];
0,0
0,1
0,2
1,0
1,1
1,2

가변 배열 (Jagged Array)

'배열의 배열'. 각 행의 열 크기가 다를 수 있어 동적인 메모리 관리에 유리합니다.

int[][] jagged = new int[3][];
[0] →
[1] →
[2] →

구조별 상대적 복잡도 및 메모리 유연성 비교

* 가변 배열은 구조적 복잡도가 높지만 메모리 낭비를 줄이는 유연성을 제공합니다.

예외 처리 및 데이터 직렬화(I/O)

프로그램의 안정성을 보장하는 예외 처리(Exception Handling) 메커니즘과, 메모리의 객체 데이터를 파일이나 네트워크용 텍스트 포맷으로 변환하는 직렬화(Serialization) 개념을 다룹니다. 직접 인터랙션 버튼을 눌러 예외 발생 시 코드 흐름이나 객체가 JSON으로 변환되는 과정을 시뮬레이션해 보세요.

예외 처리 (Try-Catch-Finally)

try {
  // 정상 코드 실행 중...
}
catch (Exception ex) {
  // 오류 처리 (대기 중)
}
finally {
  // 무조건 실행 (자원 해제)
}

직렬화 (Serialization)

Person p = new Person();
p.Name = "홍길동";
p.Age = 30;

언어 주요 특징

객체지향(OOP) 문법 및 활용

클래스, 속성, 인덱서, 그리고 다형성의 핵심인 인터페이스를 배웁니다.

핵심 문법 요소

  • 클래스(Class): 객체의 설계도 (예: Player, Monster)
  • 속성(Property): 변수처럼 보이지만 내부적으로 get/set 메서드로 동작하여 데이터 캡슐화 및 유효성 검사 수행.
  • 인덱서(Indexer): 객체를 배열처럼 `[]`로 접근하게 해주는 문법.
  • 인터페이스(Interface): 다중 상속이 가능하며, 구체적 구현 없이 '기능의 약속'만 정의하여 다형성을 제공.

실제 사용 예시 (Real-world Use Cases)

⚔️

RPG 게임 전투 시스템

`Player`와 `Monster` 클래스를 정의. 각 객체는 체력(HP) 속성을 가지며, 무기 인터페이스(`IWeapon`)를 통해 다형성(검, 총 등)을 구현하여 공격(`Attack()`) 메서드 실행.

🏠

스마트홈 기기 제어

`IPower`(전원 켜기/끄기), `IInternet`(네트워크 연결) 인터페이스를 분리. 스마트TV, 밥솥 클래스가 필요한 규약만 다중 상속받아 각각의 방식대로 기능 구현.

다차원 배열과 가변 배열

같은 자료형의 데이터를 묶어서 관리하는 구조. 차원(Dimension)에 따라 활용처가 다릅니다.

1차원 배열
int[] scores = new int[5];

단일 목록형 데이터를 저장합니다. 반복문(`for`)과 함께 사용되어 효율성을 극대화합니다.

실 사용 예시: 학생 성적 관리5명의 학생 점수를 하나의 배열에 담고, 반복문으로 총점, 평균, 최고/최저점을 계산.
2차원 배열
int[,] lotto = new int[5, 6];

행과 열로 구성된 표(Table/Grid) 형태의 데이터를 저장합니다.

실 사용 예시: 로또 번호 생성기5줄(행)의 로또를 구매할 때, 각 줄마다 6개(열)의 당첨 번호를 중복 없이 저장하고 출력.
가변 배열 (Jagged)
int[][] seats = new int[3][];

배열 안의 배열. 각 행마다 열의 길이가 다를 수 있어 메모리를 유연하게 사용합니다.

실 사용 예시: 영화관 좌석 예약1관(10석), 2관(8석), 3관(12석)처럼 상영관마다 다른 좌석 수를 가변 배열로 최적화하여 구현.

예외 처리 및 파일/데이터 관리

프로그램의 안정성을 높이고 데이터를 영구적으로 보존하는 기법입니다.

예외 처리 (try-catch-finally)

실행 중 발생하는 오류로 인한 프로그램의 강제 종료를 방지합니다.

try {
  // 오류 발생 가능 코드 (ex: 0으로 나누기)
} catch (DivideByZeroException) {
  // 예외 발생 시 처리 로직
} finally {
  // 항상 실행 (ex: ATM 카드 반환, 자원 해제)
}

파일 입출력 & JSON 직렬화

메모리의 휘발성 데이터를 `FileStream`, `StreamWriter`를 통해 하드디스크에 저장합니다.

실 사용 예시: 학생 데이터 저장/불러오기
  • 텍스트 기반(TXT): 데이터를 구분자(',')로 저장 후 분리(`Split`)하여 읽기.
  • 직렬화(JSON): `Student` 객체 자체를 표준 텍스트 포맷(JSON)으로 변환하여 통신/저장. JsonSerializer.Serialize(student) 활용.
