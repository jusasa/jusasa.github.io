# 데이터베이스 중간고사 핵심 개념 요약 (CH01 ~ CH05)

---

### 🗄️ CH01. 데이터베이스 개념

#### 데이터베이스의 4가지 정의

- 통합 데이터 (Integrated Data): 여러 곳에서 필요한 데이터를 하나로 모은(통합한) 데이터의 저장소 (중복 최소화)

- 공용 데이터 (Shared Data): 여러 사용자나 응용 프로그램들이 함께 사용할 목적으로 만든 데이터

- 운영 데이터 (Operated Data): 특정 조직의 목적을 위해 반드시 필요한 필수 데이터의 모임

- 저장 데이터 (Stored Data): 컴퓨터가 접근할 수 있는 매체에 디지털화되어 저장된 데이터

#### 데이터베이스의 4가지 특성

- 실시간 접근 (Real-time Accessibility): 다수의 사용자 요구에 수 초 안에 빠르게 응답

- 계속적인 변화 (Continuous Evolution): 현실 세계의 변화를 삽입, 삭제, 갱신을 통해 항상 최신 상태로 유지

- 동시 공유 (Concurrent Sharing): 다수의 사용자가 동시에 동일한 데이터에 접근하고 이용 가능

- 내용에 의한 참조 (Content Reference): 데이터의 물리적 위치(주소)가 아닌, 데이터가 가진 '값(내용)'으로 데이터를 검색하고 참조

#### 파일 시스템의 단점

- 데이터 종속성: 응용 프로그램과 데이터 파일이 강하게 결합되어 있어, 데이터 구조가 바뀌면 프로그램도 수정해야 함

- 데이터 중복성: 동일한 데이터가 여러 파일에 중복 저장되어 일관성 훼손 및 저장 공간 낭비 발생

### 💼 CH02. 데이터베이스 시스템

#### 데이터베이스 사용자

- DBA (데이터베이스 관리자): DB 설계, 관리, 보안, 권한 부여 등 시스템 전반을 통제하는 사람/조직

- 응용 프로그래머: 프로그래밍 언어와 DML(데이터 조작어)을 활용해 응용 프로그램을 개발하는 사람

- 일반 사용자 (최종 사용자): 제공된 UI나 질의어를 통해 DB에 접근하고 데이터를 활용하는 사람

#### 3단계 데이터베이스 구조 (스키마)

- 외부 스키마 (External Schema): 일반 사용자나 응용 프로그래머 관점에서 바라보는 개인화된 논리적 구조 (여러 개 존재 가능)

- 개념 스키마 (Conceptual Schema): 조직 전체 관점에서 데이터베이스의 전체적인 논리적 구조를 정의 (DBA가 관리, 1개만 존재)

- 내부 스키마 (Internal Schema): 물리적인 저장 장치 관점에서 데이터가 실제 어떻게 저장되는지를 정의

#### 데이터 독립성과 시스템 카탈로그

- 데이터 독립성: 하위 스키마가 변경되어도 상위 스키마에 영향을 주지 않는 성질 (논리적/물리적 데이터 독립성)

- 데이터 사전 (System Catalog / Data Dictionary): 스키마, 제약조건, 사용자 정보 등 '데이터에 관한 데이터(메타 데이터)'를 저장하는 시스템 데이터베이스

#### 데이터 언어 (SQL 분류)

- DDL (데이터 정의어): 구조를 정의/변경/삭제 (CREATE, ALTER, DROP)

- DML (데이터 조작어): 데이터를 검색/삽입/수정/삭제 (SELECT, INSERT, UPDATE, DELETE)

- DCL (데이터 제어어): 권한 부여 및 보안, 트랜잭션 제어 (GRANT, REVOKE, COMMIT, ROLLBACK)

### 📊 CH03. 관계형 데이터 모델

#### 데이터 모델의 3요소

- 데이터 구조 (Structure): 데이터를 어떻게 표현할 것인가 (예: 2차원 테이블)

- 연산 (Operation): 데이터를 어떻게 처리할 것인가 (예: 관계 대수)

- 제약 조건 (Constraint): 허용되는 데이터 값과 행위의 제한 (무결성 규칙)

#### 관계형 모델의 기본 용어

- 릴레이션 (Relation): 2차원 테이블

- 튜플 (Tuple): 테이블의 행(Row). 튜플의 총 개수 = 카디널리티 (Cardinality)

- 속성 (Attribute): 테이블의 열(Column). 속성의 총 개수 = 차수 (Degree)

- 도메인 (Domain): 하나의 속성이 가질 수 있는 원자값들의 집합

#### 키(Key)의 종류와 제약조건

- 후보키 (Candidate Key): 튜플을 유일하게 식별할 수 있는 키 (조건: 유일성 + 최소성)

- 기본키 (Primary Key): 후보키 중 선택된 메인 키

- 대체키 (Alternate Key): 기본키로 선택되지 못한 나머지 후보키

- 개체 무결성 제약조건: 기본키는 절대 NULL 값을 가질 수 없다

- 참조 무결성 제약조건: 외래키(Foreign Key) 값은 참조하는 테이블의 기본키 값이거나 NULL이어야 한다

### 🔧 CH04. 관계 대수

#### 관계 대수 개요

- 절차적 언어: 원하는 데이터를 어떻게(How) 찾을 것인지 처리 과정을 명시하는 수학적 연산 집합 (참고: 관계 해석은 '무엇(What)'만 명시하는 비절차적 언어)

#### 일반 집합 연산

- 합집합($\cup$), 교집합($\cap$), 차집합($-$): 이 세 연산을 수행하려면 두 릴레이션이 합집합 양립성(Union Compatibility) 조건을 만족해야 함 (속성 개수와 도메인이 같아야 함)

- 카티션 프로덕트($\times$): 두 릴레이션의 모든 튜플을 조건 없이 결합 (결과 튜플 수 = A 튜플 수 $\times$ B 튜플 수)

#### 순수 관계 연산

- 셀렉트 (Select, $\sigma$): 조건에 맞는 **행(튜플)**을 추출하는 단항 연산

- 프로젝트 (Project, $\pi$): 원하는 **열(속성)**만 추출하는 단항 연산

- 조인 (Join, $\bowtie$): 공통 속성을 기준으로 두 릴레이션을 결합
  - 동등 조인(Equi Join): 속성 값이 일치(=)하는 튜플만 결합
  - 자연 조인(Natural Join): 동등 조인 결과에서 중복된 속성을 제거
  - 외부 조인(Outer Join): 일치하지 않아 조인에 실패한 튜플도 널(NULL) 값을 채워 결과에 포함시킴 (LEFT, RIGHT, FULL)

### 📝 CH05. SQL 기초 (DML 중심)

#### 데이터 검색 (SELECT)

- 기본 구조: `SELECT [속성] FROM [테이블] WHERE [조건] ORDER BY [정렬기준];`

- 주요 키워드:
  - DISTINCT: 중복 결과 제거
  - LIKE: 문자열 패턴 매칭 (%는 모든 문자열, _는 단일 문자)
  - BETWEEN A AND B: A와 B 사이의 값
  - IN (A, B): A 또는 B와 일치하는 값
  - IS NULL: 널 값 확인 (주의: = NULL 은 사용 불가)

#### 그룹화 및 조인

- GROUP BY / HAVING: 특정 속성을 기준으로 그룹을 묶고(GROUP BY), 그룹에 대한 조건(HAVING)을 부여하여 집계 함수(COUNT, MAX, MIN, AVG, SUM)를 사용

- 조인 (JOIN):
  - 내부 조인 (INNER JOIN): 양쪽 테이블에 모두 존재하는 데이터만 출력
  - 외부 조인 (OUTER JOIN): 한쪽(또는 양쪽) 테이블의 데이터를 끝까지 출력 (Oracle에서는 조건절 반대편에 (+) 기호 사용)
  - 셀프 조인 (Self Join): 하나의 테이블을 두 개인 것처럼 논리적으로 분리하여 조인

#### 데이터 변경 (INSERT, UPDATE, DELETE)

- INSERT: `INSERT INTO 테이블명(열1, 열2) VALUES (값1, 값2);`

- UPDATE: `UPDATE 테이블명 SET 열 = 새값 WHERE 조건;` (WHERE 생략 시 모든 행 변경)

- DELETE: `DELETE FROM 테이블명 WHERE 조건;` (WHERE 생략 시 모든 행 삭제)
