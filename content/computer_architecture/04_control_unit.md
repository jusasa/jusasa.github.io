# 제어 유니트

## 제어 유닛의 기능

각각의 레지스터들을 켜고 끄는 (스위치), 입출력을 제어함
트랜지스터로 제어

- 마이크로 연산

- ***명령어 코드의 해독***
- ***실행에 필요한 제어신호 생성***

## 제어 유닛의 구조

| 구성 요소 | 역할 / 설명 |
| :--- | :--- |
| **명령어 해독기** | IR의 명령어를 해석<br>루틴의 시작 주소를 결정 |
| **CAR** (control address register)<br>[제어 주소 레지스터] | 다음에 실행할 마이크로 명령어의 주소를 저장하는 레지스터<br>제어 기억장치의 특정 위치를 지정 |
| **제어기억장치 ROM**<br>`~= main memory[control memory]` | 마이크로 명령어들로 이루어진 마이크로프로그램들을 저장하는 메모리 |
| **CBR** (명령어 버퍼 레지스터) | 제어 기억장치로부터 읽은 마이크로 명령어 비트들을 일시적으로 저장하는 레지스터 |
| **SBR** | 마이크로프로그램으로부터 서브루틴이 호출되는 경우 CAR 내용을 임시저장하는 레지스터 |
| **순서제어 모듈** | 마이크로 명령어들의 실행 순서를 지정해주는 모듈 |


각 명령어에 대한 프로그램 코드를 가지고 있음. (인출, 저장, 연산등)

### 명령어 세트 설계 과정

1. 명령어의 **종류** 및 **비트패턴** 정의
2. 명령어들의 실행을 위한 **하드웨어 설계**
3. 각 명령어들을 위한 실행 **사이클 루틴** 작성(마이크로 프로그래밍)
4. 마이크로프로그램 **코드들**을 **제어기억장치**에 저장.

### 명령어 해독

실행사이클 루틴의 시작주소를 결정하는 작업

* **매핑을 통한 해독방법**

    * 연산 코드를 특정 비트 패턴과 조합

## 마이크로 명령어의 형식


* 연산 필드가 2개이면 2개의 마이크로 연산을 동시에 처리 가능
* CD 필드는 분기 조건 플래그
* BR은 분기 주소

>  예시
 $\text{|연산 필드1|연산필드2|조건필드(CD)|분기필드(BR)|주소필드(ADF)|}$

## 마이크로 프로그램의 순서제어

* CAR의 초기값: 0
